# Nilesh Rane Portfolio

A smart, responsive, and interactive portfolio website built from resume content and a profile image.

## Overview

This portfolio presents Nilesh Rane as a senior safety professional with strong experience in:

- Safety leadership
- Elevator operations
- Site compliance
- Workforce coordination
- Installation and repair support

The site is built as a lightweight static website, so it can be opened directly in a browser or deployed easily on GitHub Pages.

## Project Files

- `index.html` - Main website structure and content
- `styles.css` - Visual design, responsive layout, hover effects, and animations
- `script.js` - Interactive behavior like menu toggle, section reveal, counters, filters, and card hover motion
- `profile.jpeg` - Profile image used in the hero section
- `resume.docx` - Resume file used for download

## Features

- Responsive portfolio layout
- Premium hero section with profile image
- Experience timeline
- Skill filter and animated progress bars
- Smooth reveal animations
- Smart hover highlight and pop-up card effects
- LinkedIn call-to-action
- Downloadable resume

## Run Locally

You do not need any installation for this project.

1. Download or clone the repository
2. Open `index.html` in your browser

## Upload To GitHub

1. Create a new GitHub repository
2. Upload all project files from this folder
3. Make sure these files are included:

- `index.html`
- `styles.css`
- `script.js`
- `profile.jpeg`
- `resume.docx`
- `README.md`

## Make It Live With GitHub Pages

1. Open your repository on GitHub
2. Go to `Settings`
3. Open `Pages`
4. Under `Build and deployment`, choose:

- `Source`: `Deploy from a branch`
- `Branch`: `main`
- `Folder`: `/ (root)`

5. Click `Save`

After GitHub finishes deployment, your site will be live on a URL like:

`https://your-username.github.io/your-repository-name/`

## Important Notes

- Keep file names unchanged unless you also update the references inside `index.html`
- If you replace the image, keep the same file name `profile.jpeg` or update the image path
- If you replace the resume, keep the same file name `resume.docx` or update the resume link

## Customization

You can easily update:

- Name and role in `index.html`
- About and experience content in `index.html`
- Colors, spacing, and hover styles in `styles.css`
- Interactive behavior in `script.js`

## Deployment Tip

If GitHub Pages does not update immediately, wait a minute and refresh the published URL. Sometimes the first deployment takes a little time.
